## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка переменных окружения

```bash
cp .env.automation.example .env.local
# Отредактируйте .env.local
```

### 3. Создание 20 сателлитов

```bash
# Полный цикл: генерация → сборка → деплой → SEO
node scripts/master.js --generate 20 --niche crypto --pages 500
```

### 4. Миграция существующих сателлитов

```bash
# Мигрировать на новую архитектуру
node scripts/master.js --migrate /path/to/your/satellites
```

## 🏗️ Архитектура системы

```
scripts/
├── master.js                # 🎯 Главный оркестратор
├── satellite-generator.js   # 🛠️ Генератор одного сателлита

├── migrate-satellite.js     # 🔄 Миграция одного сателлита

├── deploy-all.js            # ☁️ Деплой на Vercel
└── submit-to-search.js      # 🔍 Отправка в поисковики
```

```
┌─────────────────┐

         ▼

         │
         ▼

         │
         ▼

└─────────────────┘
```

## 📖 Сценарии использования

### Сценарий 1: Создание 20 новых сателлитов

```bash
# Полный автоматический цикл
node scripts/master.js --generate 20 --niche crypto --pages 500

# С разными нишами (random выбирает случайно)
node scripts/master.js --generate 20 --niche random --pages 1000

# Только генерация и сборка (без деплоя)
node scripts/master.js --generate 20 --skip-deploy
```

**Результат:**
- 20 новых проектов в `../satellites/`
- Каждый с ~500 уникальными страницами
- Автоматически собраны и задеплоены на Vercel
- Sitemap отправлены в поисковики

**Время выполнения:** ~30-60 минут (зависит от мощности ПК)

---

### Сценарий 2: Миграция существующих сателлитов

У вас уже есть сателлиты на старой архитектуре? Мигрируйте их:

```bash
# Мигрировать все сателлиты из директории
node scripts/master.js --migrate ~/projects/my-satellites

# Только миграция (без деплоя)
node scripts/master.js --migrate ~/projects/my-satellites --skip-deploy
```

**Что происходит:**
1. Создаётся бэкап старых данных
2. Монолитный `seo-data.js` разбивается на модули
3. Копируются новые компоненты (PerformanceMonitor, RSS, и т.д.)
4. Обновляются импорты
5. Пересборка и деплой

---

### Сценарий 3: Только деплой

Сателлиты уже созданы и собраны, нужно только задеплоить:

```bash
node scripts/master.js --deploy-only
```

---

### Сценарий 4: Пошаговый контроль

Если нужен полный контроль на каждом этапе:

```bash
# Шаг 1: Генерация
node scripts/batch-generator.js --count 20 --niche crypto --pages 500

# Шаг 2: Сборка
node scripts/build-all.js

# Шаг 3: Деплой
node scripts/deploy-all.js

# Шаг 4: SEO
node scripts/submit-to-search.js
```

## 🔑 Настройка API ключей

### Vercel (обязательно для автоматического деплоя)

```bash
# 1. Получите токен: https://vercel.com/account/tokens
export VERCEL_TOKEN=your_vercel_token_here

# 2. (Опционально) ID команды для Vercel Pro
export VERCEL_TEAM=team_xxxxxxxxxxxxx
```

### Яндекс Вебмастер (для автоматической отправки sitemap)

```bash
# 1. Получите токен: https://oauth.yandex.ru/
export YANDEX_WEBMASTER_TOKEN=your_token_here

# 2. Получите User ID из Яндекс Вебмастер
export YANDEX_USER_ID=your_user_id
```

### Google Search Console

Google Indexing API требует OAuth2 и сложную настройку. Рекомендуем отправлять sitemap вручную:

1. Откройте [Google Search Console](https://search.google.com/search-console)
2. Добавьте каждый сайт
3. Перейдите в "Файлы Sitemap"
4. Добавьте `sitemap.xml`

## 💎 Vercel Pro оптимизация

У вас Vercel Pro? Отлично! Вот как максимально использовать его:

### Преимущества Vercel Pro

✅ **Бесплатные домены** — `your-site.vercel.app` без ограничений
✅ **Unlimited deployments** — деплойте сколько угодно
✅ **100 GB bandwidth/месяц** — достаточно для старта
✅ **Приоритетная сборка** — быстрее деплой

### Рекомендации

1. **Используйте Vercel домены для старта**
   - Не покупайте домены сразу
   - Протестируйте сателлиты на `*.vercel.app`
   - Купите домены только для успешных сателлитов

2. **Организация проектов**
   ```bash
   # Создайте команду в Vercel для всех сателлитов
   export VERCEL_TEAM=team_your_team_id

   # Все сателлиты будут в одной команде
   node scripts/master.js --generate 20
   ```

3. **Мониторинг трафика**
   - Следите за Analytics в Vercel Dashboard
   - Если сателлит набирает трафик → купите домен
   - Если нет трафика → удалите через 1-2 месяца

4. **Лимиты**
   - 100 GB bandwidth/месяц на команду
   - ~500 страниц × 20 сателлитов = 10,000 страниц
   - При 100 KB/страница = 1 GB на полный обход
   - **Вывод:** можно обслужить ~100 полных обходов/месяц

## 📈 Масштабирование

### Создание 20 сателлитов в день

**Технически возможно:**
```bash
# Утром: 20 сателлитов
node scripts/master.js --generate 20 --niche crypto --pages 500

# Вечером: ещё 20 сателлитов
node scripts/master.js --generate 20 --niche fitness --pages 500
```

**Но есть ограничения:**

1. **Контент должен быть уникальным**
   - Используйте разные ниши
   - Меняйте структуру данных
   - Генерируйте уникальные тексты (GPT-4/Claude)

2. **Vercel лимиты**
   - 100 GB bandwidth/месяц (Pro)
   - Unlimited deployments ✅
   - Но следите за трафиком

3. **Google фильтры**
   - Thin content → бан
   - Дублированный контент → бан
   - Массовые сателлиты с одного IP → подозрительно

### Рекомендуемая стратегия

**Вместо 20 сателлитов/день:**

```
Неделя 1: 5 сателлитов (тестирование)
Неделя 2: 10 сателлитов (если первые работают)
Неделя 3: 20 сателлитов (масштабирование)
Неделя 4: Анализ результатов
```

**Критерии успеха:**
- Индексация в Google (проверка: `site:your-domain.vercel.app`)
- Трафик из поиска (Google Analytics)
- Нет фильтров/банов

## ⚠️ Важные предупреждения

### 1. Качество контента

❌ **НЕ ДЕЛАЙТЕ:**
- Копирование контента с других сайтов
- Спин контента (автоматическая перефразировка)
- Тонкий контент (<300 слов на страницу)

✅ **ДЕЛАЙТЕ:**
- Уникальный контент для каждой страницы
- Полезная информация для пользователей
- Минимум 500-1000 слов на важных страницах

### 2. Скорость генерации

❌ **НЕ ДЕЛАЙТЕ:**
- 100+ сателлитов за день
- Все сателлиты с одинаковым контентом
- Массовая отправка в Google Indexing API

✅ **ДЕЛАЙТЕ:**
- Постепенное масштабирование (5 → 10 → 20)
- Разные ниши и структуры
- Естественная индексация (через sitemap)

### 3. Vercel лимиты

Следите за:
- Bandwidth (100 GB/месяц на Pro)
- Build minutes (6000 мин/месяц на Pro)
- Serverless function executions
