# ❓ FAQ — Частые вопросы

## 📋 Общие вопросы

### Что такое SEO-сателлит?

SEO-сателлит — это дополнительный сайт, который создаётся для привлечения трафика из поисковых систем и перенаправления его на основной (money) сайт. Сателлиты обычно содержат информационный контент, оптимизированный под низкочастотные запросы.

---

### Зачем нужна автоматизация?

Ручное создание сателлитов занимает много времени:
- Создание проекта: 30-60 минут
- Настройка данных: 1-2 часа
- Деплой: 10-20 минут
- SEO настройка: 30 минут

**Итого:** 2-4 часа на один сателлит.

С автоматизацией:
- 20 сателлитов за 60-90 минут
- Всё автоматически: генерация → сборка → деплой → SEO

---

### Это легально?

Да, создание информационных сайтов легально. Но:

✅ **Легально:**
- Уникальный контент
- Полезная информация для пользователей
- Естественные ссылки на основной сайт

❌ **Нелегально/Рискованно:**
- Копирование контента
- Спам-ссылки
- Обман пользователей
- Thin content (мало текста)

---

### Можно ли использовать бесплатно?

Да! Если у вас Vercel Pro:
- ✅ Бесплатные домены (`*.vercel.app`)
- ✅ Unlimited deployments
- ✅ 100 GB bandwidth/месяц

Дополнительные затраты (опционально):
- Домены: $10-15/год за домен
- API для генерации контента: $10-50/месяц

---

### Сколько времени до первых результатов?

**Индексация:**
- Google: 3-14 дней
- Яндекс: 1-7 дней

**Трафик:**
- Первые посетители: 1-2 недели
- Стабильный трафик: 1-3 месяца
- Значимый трафик: 3-6 месяцев

## 🛠️ Технические вопросы

### Какие требования к системе?

**Минимальные:**
- Node.js >= 18.x
- 4 GB RAM
- 10 GB свободного места

**Рекомендуемые:**
- Node.js >= 20.x
- 8 GB RAM
- 50 GB свободного места (для 50+ сателлитов)

---

### Работает ли на Windows?

Да! Все скрипты кроссплатформенные:
- ✅ Windows 10/11
- ✅ macOS
- ✅ Linux
- ✅ WSL (Windows Subsystem for Linux)

---

### Нужен ли мощный компьютер?

Нет, но:
- **Генерация:** Лёгкая (любой ПК)
- **Сборка:** Средняя (4-8 GB RAM)
- **Деплой:** Лёгкая (зависит от интернета)

Для 20 сателлитов рекомендуется:
- 8 GB RAM
- SSD диск
- Стабильный интернет

---

### Можно ли запускать на сервере?

Да! Идеально для автоматизации:

```bash
# На сервере (Linux)
crontab -e

# Добавьте:
0 9 * * * cd /path/to/project && npm run satellite:master -- --generate 5
```

Это будет создавать 5 сателлитов каждый день в 9:00.

---

### Как обновить существующие сателлиты?

```bash
# Мигрировать на новую архитектуру
npm run satellite:migrate-all -- --dir /path/to/satellites

# Пересобрать
npm run satellite:build-all

# Задеплоить
npm run satellite:deploy-all
```

## 🔍 SEO вопросы

### Как быстро индексируется сайт?

**Google:**
- Отправка sitemap: мгновенно
- Первая индексация: 3-14 дней
- Полная индексация: 2-4 недели

**Яндекс:**
- Отправка sitemap: мгновенно
- Первая индексация: 1-7 дней
- Полная индексация: 1-2 недели

**Ускорение:**
- Отправьте sitemap в Search Console
- Запросите индексацию главной страницы
- Добавьте ссылки с других сайтов

---

### Нужно ли покупать домены?

**Нет, не обязательно!**

Vercel Pro даёт бесплатные домены:
- `your-site.vercel.app`
- Полностью функциональные
- Индексируются Google/Яндекс

**Когда покупать домены:**
- Сателлит набрал трафик (>100 посетителей/день)
- Хотите брендированный домен
- Нужна дополнительная доверенность

---

### Как избежать фильтров Google?

✅ **Делайте:**
- Уникальный контент (минимум 500 слов)
- Полезная информация для пользователей
- Естественные ссылки
- Разные ниши и структуры
- Постепенное масштабирование

❌ **Не делайте:**
- Копирование контента
- Спин-контент
- Thin content (<300 слов)
- Массовые сателлиты за день
- Одинаковый контент на всех сателлитах

---

### Сколько сателлитов можно создать?

**Технически:** Неограниченно (с Vercel Pro)

**Практически:**
- Начните с 5-10
- Проверьте индексацию
- Масштабируйте до 20-50
- Следите за качеством

**Лимиты Vercel Pro:**
- Bandwidth: 100 GB/месяц
- Deployments: Unlimited
- Build minutes: 6000/месяц

**Расчёт:**
- 50 сателлитов × 500 страниц = 25,000 страниц
- При 100 KB/страница = 2.5 GB на полный обход
- **Вывод:** 100 GB хватит на ~40 полных обходов/месяц

## 🔧 Вопросы о настройке

### Как получить Vercel токен?

1. Откройте https://vercel.com/account/tokens
2. Нажмите "Create Token"
3. Введите название (например, "Satellites Automation")
4. Выберите Scope: "Full Account"
5. Нажмите "Create"
6. Скопируйте токен
7. Экспортируйте:
   ```bash
   export VERCEL_TOKEN=your_token_here
   ```

---

### Как получить Vercel Team ID?

1. Откройте https://vercel.com/teams
2. Выберите команду
3. ID команды в URL: `vercel.com/teams/TEAM_ID`
4. Скопируйте TEAM_ID
5. Экспортируйте:
   ```bash
   export VERCEL_TEAM=team_xxxxxxxxxxxxx
   ```

---

### Как получить Yandex Webmaster токен?

1. Откройте https://oauth.yandex.ru/
2. Нажмите "Зарегистрировать приложение"
3. Заполните форму:
   - Название: "Satellites SEO"
   - Права: "Яндекс.Вебмастер"
4. Получите Client ID и Client Secret
5. Получите токен через OAuth flow
6. Экспортируйте:
   ```bash
   export YANDEX_WEBMASTER_TOKEN=your_token
   ```

## 🆘 Вопросы о проблемах

### Что делать, если сателлит не индексируется?

1. **Проверьте robots.txt:**
   ```
   https://your-domain.vercel.app/robots.txt
   ```
   Должен разрешать индексацию.

2. **Проверьте sitemap:**
   ```
   https://your-domain.vercel.app/sitemap.xml
   ```
   Должен быть доступен и корректен.

3. **Отправьте в Search Console:**
   - Добавьте сайт
   - Отправьте sitemap
   - Запросите индексацию

4. **Подождите:**
   - Google: 3-14 дней
   - Яндекс: 1-7 дней

5. **Проверьте качество:**
   - Уникальный контент?
   - Достаточно текста (>500 слов)?
   - Нет дублей?

---

### Что делать при ошибке "Bandwidth exceeded"?

1. **Проверьте использование:**
   - Vercel Dashboard → Usage

2. **Оптимизируйте:**
   - Удалите неиспользуемые сателлиты
   - Оптимизируйте изображения
   - Включите кэширование

3. **Апгрейд:**
   - Vercel Pro → Enterprise
   - Или используйте CDN
