# План миграции с Cloudflare на Vercel

## Причины миграции

1. **Доступность в РФ**: Cloudflare может быть заблокирован из-за санкций, Vercel работает стабильно
2. **Оптимизация затрат**: Vercel Pro план с 6,000 build минут/месяц
3. **Простота деплоя**: Vercel CLI + GitHub integration

## Текущая ситуация

- **Cloudflare Workers**: 27 сателлитов задеплоено
- **Страниц**: 10,235 страниц LIVE
- **Проблема**: Доступность в РФ под вопросом

## Целевая архитектура на Vercel

### Оптимизация под Vercel Pro

**Лимиты Vercel Pro:**
- Build минут: 6,000/месяц
- Bandwidth: 1TB/месяц
- Serverless функций: 1,000,000/месяц

**Стратегия оптимизации:**

1. **Сократить количество сателлитов в день**
   - Было: 20 сателлитов/день
   - Будет: 5-10 сателлитов/день
   - Причина: экономия build минут

2. **Расчет build минут**
   ```
   1 сателлит = ~2-3 минуты build
   5 сателлитов/день × 30 дней = 150 сателлитов/месяц
   150 × 3 минуты = 450 минут/месяц
   
   Запас: 6,000 - 450 = 5,550 минут (92% остается)
   ```

3. **Увеличить страниц на сателлит**
   - Было: 300-1000 страниц
   - Будет: 1000-2000 страниц
   - Результат: тот же объем контента, меньше build минут

## Технические изменения

### 1. Новые скрипты

**scripts/vercel-deploy-all.js**
- Деплой всех сателлитов на Vercel
- Параллельность: 3 сателлита одновременно
- Трекинг build минут
- Кеширование сборок

**scripts/vercel-daily-generator.js**
- Ежедневная генерация 5-10 сателлитов
- Оптимизирован под Vercel Pro
- Интеграция с Vercel CLI

### 2. GitHub Actions

**.github/workflows/vercel-daily-generation.yml**
- Замена Cloudflare workflow
- Расписание: 6:00 МСК ежедневно
- По умолчанию: 5 сателлитов (вместо 20)
- Автоматический деплой на Vercel

### 3. Конфигурация

**vercel.json** (уже существует)
- Headers для SEO
- Redirects
- ISR настройки

**package.json** - новые команды:
```json
{
  "satellite:vercel-deploy": "node scripts/vercel-deploy-all.js",
  "satellite:vercel-daily": "node scripts/vercel-daily-generator.js"
}
```

## Процесс миграции

### Этап 1: Подготовка (5 минут)

1. Создать новые скрипты для Vercel
2. Создать новый GitHub Actions workflow
3. Обновить package.json

### Этап 2: Настройка Vercel (10 минут)

1. Получить Vercel Token: https://vercel.com/account/tokens
2. Добавить в GitHub Secrets:
   - `VERCEL_TOKEN`
   - `VERCEL_ORG_ID` (опционально)

### Этап 3: Тестовый деплой (15 минут)

1. Локально сгенерировать 1 тестовый сателлит
2. Задеплоить на Vercel вручную
3. Проверить работоспособность из РФ

### Этап 4: Автоматизация (5 минут)

1. Активировать новый GitHub Actions workflow
2. Отключить старый Cloudflare workflow
3. Запустить первую автоматическую генерацию

### Этап 5: Мониторинг (ongoing)

1. Отслеживать использование build минут
2. Проверять доступность из РФ
3. Мониторить трафик и SEO метрики

## Рекомендации

### Оптимальная стратегия для Vercel Pro

1. **Ежедневная генерация: 5 сателлитов**
   - 5 × 30 дней = 150 сателлитов/месяц
   - 150 × 1000 страниц = 150,000 страниц/месяц
   - Build минут: ~450/месяц (7.5% от лимита)

2. **Страниц на сателлит: 1000-2000**
   - Больше контента на сателлит
   - Меньше build операций
   - Эффективнее использование ресурсов

3. **Кеширование и ISR**
   - ISR: revalidate каждые 6 часов
   - Static pages: кешируются навсегда
   - API routes: on-demand revalidation

4. **Мониторинг использования**
   - Vercel Dashboard: отслеживать build минуты
   - GitHub Actions: логи деплоя
   - Alerts: уведомления при превышении 80% лимита
